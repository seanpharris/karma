# Theme bundles

A "theme" picks one variant per layer category to compose a complete
character. Drop JSON bundles in this folder; the (TBD) Karma-side LPC
loader walks each bundle, resolves the layer paths into
`../spritesheets/`, and composites them into a runtime sheet.

Bundle shape (proposed):

```json
{
  "id": "fantasy_warrior_male",
  "display_name": "Fantasy Warrior",
  "tags": ["fantasy", "law_aligned", "melee"],
  "layers": {
    "body":     "body/bodies/male/",
    "feet":     "feet/boots/leather/male/",
    "legs":     "legs/pants/pants_brown/male/",
    "torso":    "torso/clothes/longsleeve/longsleeve_brown/male/",
    "head":     "head/heads/human_male/",
    "hair":     "hair/short/brown/male/",
    "weapon":   "weapon/sword/longsword/foreground/"
  },
  "animations": ["walk", "idle", "slash"]
}
```

Resolution is `<layer>: <path under spritesheets/>`. The loader finds
`<path>/<animation>.png` for each requested animation and stacks them in
LPC z-order (body first, weapon last).

Conventions:
- `id` is a stable string used as the runtime key.
- `tags` are free-form for filtering (theme-pick UI, NPC role pools, etc.).
- `layers` only includes the categories you want — omitted layers don't
  render. A naked body with no clothing is a valid bundle.
- `animations` is a hint for which sheets to pre-compose; the loader
  should still gracefully handle missing per-variant animations
  (each variant's `sheet_definitions/<category>/<variant>.json` lists what
  it actually ships).

Don't write themes by hand for every NPC — once we have a few canonical
ones (e.g. fantasy_warrior_male, scifi_engineer_female, modern_civilian),
NPCs reference them by id.

## Medieval generation

Karma's medieval bundle set is generated by
`tools/lpc_generate_medieval_bundles.py`, then materialized into runtime
atlases with `tools/lpc_materialize_theme_bundles.gd`.

The medieval generator scans the local LPC spritesheet tree and creates 24
deterministic variants for each base NPC role. It intentionally uses broad
clothing, hair, armor, weapon, accessory, and color pools, including
`walk/<color>.png` sheets where LPC provides them. It also emits skin,
hair, and beard tint hints that the materializer applies while composing
the final PNGs.

The scanner filters out non-medieval paths such as glowsword, laser,
firearm, sci-fi, modern, robot, space, and holiday assets.
